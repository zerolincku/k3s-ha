# 在线一键部署

本文说明如何从一台运维机通过 SSH 部署 3 master K3s HA 集群。

## 部署脚本

脚本位置：

```bash
k3s/scripts/deploy-k3s-ha.sh
```

脚本会完成：

- 校验三台 master 的 SSH 连通性。
- 自动识别每台 master 的 CPU 架构与 cgroup 模式。
- 在三台机器上写入 K3s 所需 sysctl 配置。
- 在线安装 `curl` 等 K3s 依赖。
- 在线下载并安装 K3s。
- master1 使用 `--cluster-init` 初始化 embedded etcd。
- master2、master3 默认通过 `https://MASTER1_HOST:6443` 加入。
- 拉取 kubeconfig 并将 server 地址改为 `KUBECONFIG_SERVER`。
- 执行基础健康检查。

## 配置 inventory

复制模板：

```bash
cp k3s/配置示例.env k3s/prod.env
```

按真实环境修改：

```bash
SSH_USER=root
MASTER1_HOST=192.168.1.11
MASTER1_ARCH=auto
MASTER2_HOST=192.168.1.12
MASTER2_ARCH=auto
MASTER3_HOST=192.168.1.13
MASTER3_ARCH=auto
K3S_JOIN_ENDPOINT=
KUBECONFIG_SERVER=
TLS_SAN_VALUES=
CGROUP_MODE=auto
```

生产环境建议显式固定：

```bash
K3S_VERSION=v1.34.8+k3s1
K3S_TOKEN=<使用 openssl rand -hex 32 生成>
```

如果不设置 `K3S_VERSION`，脚本会使用 K3s install script 的默认 stable channel。生产环境不建议这样做。

如果希望强制三台 master 都是 cgroup v2：

```bash
CGROUP_MODE=v2
```

如果某台机器不匹配，脚本会在部署前中止。

如果留空：

- `K3S_JOIN_ENDPOINT` 默认是 `https://MASTER1_HOST:6443`。
- `KUBECONFIG_SERVER` 默认使用 `K3S_JOIN_ENDPOINT`。
- `TLS_SAN_VALUES` 不写额外 SAN。

如果你已经知道未来 API VIP，可以提前写：

```bash
TLS_SAN_VALUES=192.168.1.10
```

K3s 部署完成后，可以再单独执行 `keepalived-haproxy/`，然后把 kubeconfig 的 server 改成 `https://VIP:8443`。

## 执行部署

```bash
bash k3s/scripts/deploy-k3s-ha.sh k3s/prod.env
```

部署完成后，本地会生成：

```text
./k3s/artifacts/kubeconfig.yaml
```

使用 kubeconfig：

```bash
export KUBECONFIG=./k3s/artifacts/kubeconfig.yaml
kubectl get nodes -o wide
```

## 幂等性边界

脚本面向新集群部署。它可以重复写入 K3s 系统参数和配置，但不会自动销毁已有 K3s 数据目录。

如果目标机器已经部署过 K3s，需要人工确认是否保留数据：

```bash
systemctl status k3s
ls -lah /var/lib/rancher/k3s
```

确认要重装时，先在每台机器执行官方卸载脚本：

```bash
/usr/local/bin/k3s-uninstall.sh
```

生产环境不要在未备份 etcd snapshot 的情况下执行卸载。

## 验证

```bash
kubectl get nodes -o wide
kubectl get nodes -L kubernetes.io/arch,node.k3s-ha.io/cgroup-version
kubectl get --raw=/readyz?verbose
kubectl -n kube-system get pods -o wide
```

检查 etcd 成员：

```bash
sudo k3s etcd-snapshot ls
sudo k3s kubectl -n kube-system get endpoints kubernetes
```

检查当前 kubeconfig 指向的 API 入口：

```bash
kubectl get --raw=/readyz
```

如果后续部署了 VIP/HAProxy，再额外检查：

```bash
curl -k https://<VIP>:8443/readyz
```
