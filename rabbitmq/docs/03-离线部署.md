# 离线部署

RabbitMQ 离线部署需要准备：

```text
RabbitMQ Cluster Operator manifest
RabbitMQ Cluster Operator image
RabbitMQ server image
```

K3s 本身离线部署由 `k3s/` 目录描述，本目录只描述 RabbitMQ 资源。

## 下载离线资源

在有网络的机器执行：

```bash
cp rabbitmq/config.example.env rabbitmq/prod.env
vim rabbitmq/prod.env

bash rabbitmq/scripts/download-rabbitmq-images.sh rabbitmq/prod.env
```

默认输出：

```text
rabbitmq/artifacts/operator/cluster-operator-v2.21.0.yml
rabbitmq/artifacts/images/rabbitmq-4.3.1-management.tar
rabbitmq/artifacts/images/rabbitmqoperator-cluster-operator-2.21.0.tar
```

如果需要代理，先导出：

```bash
export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890
export all_proxy=socks5://127.0.0.1:7890
```

## 上传到 K3s 自动导入目录

K3s 会从每个节点的以下目录导入镜像归档：

```text
/var/lib/rancher/k3s/agent/images/
```

配置节点：

```bash
RABBITMQ_NODE_SSH_HOSTS=k3s-master-1,k3s-master-2,k3s-master-3
RABBITMQ_NODE_NAMES=k3s-master-1,k3s-master-2,k3s-master-3
SSH_USER=root
```

上传镜像：

```bash
bash rabbitmq/scripts/upload-rabbitmq-images.sh rabbitmq/prod.env
```

如果 K3s 已经在运行，只把 tar 放进 images 目录不会立即完成导入。需要重启对应节点的 K3s 服务，才会立刻触发自动导入。生产环境建议逐台滚动重启：

```bash
RABBITMQ_RESET_K3S_IMAGE_CACHE=true \
RABBITMQ_RESTART_K3S_AFTER_IMAGE_UPLOAD=true \
bash rabbitmq/scripts/upload-rabbitmq-images.sh rabbitmq/prod.env
```

## 离线部署

使用本地 Operator manifest：

```bash
RABBITMQ_OPERATOR_MANIFEST=rabbitmq/artifacts/operator/cluster-operator-v2.21.0.yml \
bash rabbitmq/scripts/deploy-rabbitmq.sh rabbitmq/prod.env
```

部署脚本不会下载二进制。只要节点已导入 Operator 和 RabbitMQ 镜像，Pod 就可以离线启动。

## 离线检查

确认镜像已导入：

```bash
k3s ctr images ls | grep rabbitmq
k3s ctr images ls | grep cluster-operator
```

确认 Pod 没有外部拉镜像失败：

```bash
kubectl -n rabbitmq describe pod rabbitmq-server-0 | grep -E 'Image|Pull|Err|BackOff'
kubectl -n rabbitmq-system get pods
```
