# 在线部署

## 前置条件

需要已经可用的 K3s 集群：

```bash
kubectl get nodes -o wide
kubectl get storageclass
```

确认存在 `local-path` StorageClass：

```bash
kubectl get storageclass local-path
```

## 配置

复制配置：

```bash
cp rabbitmq/config.example.env rabbitmq/prod.env
```

按环境修改：

```bash
KUBECONFIG=./k3s/artifacts/kubeconfig.yaml
RABBITMQ_NAMESPACE=rabbitmq
RABBITMQ_CLUSTER_NAME=rabbitmq
RABBITMQ_REPLICAS=3
RABBITMQ_IMAGE=rabbitmq:4.3.1-management
RABBITMQ_STORAGE_CLASS=local-path
RABBITMQ_STORAGE=20Gi
```

生产环境要固定镜像版本，不使用 `latest`。

## 执行部署

```bash
bash rabbitmq/scripts/deploy-rabbitmq.sh rabbitmq/prod.env
```

脚本会完成：

- 安装 RabbitMQ Cluster Operator。
- 创建 RabbitMQ namespace。
- 渲染并应用 `RabbitmqCluster`。
- 等待 StatefulSet rollout。
- 打印 Pod、PVC、Service 状态。

## 验证

查看 Pod 分布：

```bash
kubectl -n rabbitmq get pods -o wide
```

查看 PVC/PV：

```bash
kubectl -n rabbitmq get pvc
kubectl get pv
```

查看 PV 节点亲和：

```bash
pv=$(kubectl -n rabbitmq get pvc -o jsonpath='{.items[0].spec.volumeName}')
kubectl get pv "$pv" -o yaml | grep -A20 nodeAffinity
```

查看集群状态：

```bash
kubectl -n rabbitmq exec rabbitmq-server-0 -- rabbitmq-diagnostics cluster_status
```

查看 quorum 状态：

```bash
kubectl -n rabbitmq exec rabbitmq-server-0 -- rabbitmq-queues quorum_status --all
```

获取默认管理员账号：

```bash
kubectl -n rabbitmq get secret rabbitmq-default-user -o yaml
```

具体 secret 名称以 Operator 生成结果为准：

```bash
kubectl -n rabbitmq get secret
```
