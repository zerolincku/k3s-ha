# 在线部署

## 前置条件

需要已经可用的 K3s 集群：

```bash
kubectl get nodes -o wide
kubectl -n kube-system get pods
```

确认三台 K3s master 可以调度业务 Pod。如果控制面节点带 taint，本方案 manifest 已包含常见 toleration。

## 配置

复制配置：

```bash
cp redis/config.example.env redis/prod.env
```

修改密码和镜像：

```bash
REDIS_IMAGE=redis:7.2.4
REDIS_PASSWORD=<使用 openssl rand -base64 24 生成>
```

生产环境不要使用 `latest` 镜像，也不要保留示例密码。默认使用 `redis:7.2.4` 是一个保守示例；实际生产应按公司许可、漏洞基线和内网镜像仓库情况固定镜像。

## 执行部署

```bash
bash redis/scripts/deploy-redis-sentinel.sh redis/prod.env
```

脚本会完成：

- 创建 namespace。
- 创建或更新 `redis-auth` Secret。
- 渲染并应用 Redis Sentinel manifest。
- 等待 `redis` 和 `sentinel` 两个 StatefulSet 完成 rollout。
- 打印 Redis、Sentinel Pod 状态。

## 验证

查看 Pod：

```bash
kubectl -n redis get pods -o wide
```

查看 Sentinel 发现的 master：

```bash
kubectl -n redis exec sentinel-0 -- \
  redis-cli -p 26379 sentinel get-master-addr-by-name mymaster
```

测试 Redis 认证：

```bash
password=$(kubectl -n redis get secret redis-auth -o jsonpath='{.data.password}' | base64 -d)

kubectl -n redis exec redis-0 -- \
  redis-cli -a "$password" --no-auth-warning ping
```

查看主从角色：

```bash
for pod in redis-0 redis-1 redis-2; do
  kubectl -n redis exec "$pod" -- \
    sh -c 'redis-cli -a "$REDIS_PASSWORD" --no-auth-warning role | head -n1'
done
```

## 客户端连接

应用连接 Sentinel Service：

```text
redis-sentinel.redis.svc.cluster.local:26379
```

参数：

```text
master name: mymaster
password: redis-auth Secret 中的 password
```

不同语言客户端参数名不同，核心要求是启用 Sentinel 模式，而不是直接写死 `redis-0`。
