# 运维手册

## 常用状态检查

查看 Rook/Ceph Pod：

```bash
kubectl -n rook-ceph get pods -o wide
```

查看 Ceph 自定义资源：

```bash
kubectl -n rook-ceph get cephcluster,cephfilesystem,cephobjectstore
```

查看 StorageClass：

```bash
kubectl get storageclass | grep rook-ceph
```

## 使用 Ceph 工具箱

部署工具箱：

```bash
kubectl -n rook-ceph apply -f https://raw.githubusercontent.com/rook/rook/v1.19.6/deploy/examples/toolbox.yaml
kubectl -n rook-ceph rollout status deploy/rook-ceph-tools --timeout=300s
```

进入：

```bash
kubectl -n rook-ceph exec -it deploy/rook-ceph-tools -- bash
```

常用命令：

```bash
ceph status
ceph health detail
ceph osd tree
ceph osd df
ceph df
ceph fs status
```

## 健康状态含义

```text
HEALTH_OK
  正常。

HEALTH_WARN
  有警告，需要看 detail。常见是容量接近阈值、恢复中、某些 daemon 重启。

HEALTH_ERR
  严重异常，需要停止扩容/升级/大规模写入，先处理存储健康。
```

查看详情：

```bash
kubectl -n rook-ceph exec -it deploy/rook-ceph-tools -- ceph health detail
```

## 容量检查

```bash
kubectl -n rook-ceph exec -it deploy/rook-ceph-tools -- ceph df
kubectl -n rook-ceph exec -it deploy/rook-ceph-tools -- ceph osd df
```

容量规划建议：

```text
长期使用不要超过 70%
接近 80% 必须扩容或清理
不要等到 nearfull / full 再处理
```

## CephFS 检查

查看文件系统：

```bash
kubectl -n rook-ceph exec -it deploy/rook-ceph-tools -- ceph fs status
```

查看 MDS Pod：

```bash
kubectl -n rook-ceph get pods -l app=rook-ceph-mds -o wide
```

测试 RWX PVC：

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: cephfs-test
spec:
  accessModes:
    - ReadWriteMany
  storageClassName: rook-cephfs
  resources:
    requests:
      storage: 1Gi
```

## RGW 检查

查看 RGW Pod：

```bash
kubectl -n rook-ceph get pods -l app=rook-ceph-rgw -o wide
```

查看 RGW Service：

```bash
kubectl -n rook-ceph get svc | grep rook-ceph-rgw
```

默认集群内 endpoint：

```text
http://rook-ceph-rgw-objectstore.rook-ceph.svc:80
```

读取默认用户密钥：

```bash
kubectl -n rook-ceph get secret rook-ceph-object-user-objectstore-admin \
  -o jsonpath='{.data.AccessKey}' | base64 -d; echo

kubectl -n rook-ceph get secret rook-ceph-object-user-objectstore-admin \
  -o jsonpath='{.data.SecretKey}' | base64 -d; echo
```

## Bucket 管理

业务推荐通过 ObjectBucketClaim 创建 bucket：

```yaml
apiVersion: objectbucket.io/v1alpha1
kind: ObjectBucketClaim
metadata:
  name: app-bucket
  namespace: default
spec:
  generateBucketName: app
  storageClassName: rook-ceph-bucket
```

查看：

```bash
kubectl get obc -A
kubectl get ob -A
```

## Dashboard

默认启用 Ceph Dashboard，HTTP 模式：

```bash
kubectl -n rook-ceph get svc rook-ceph-mgr-dashboard
```

临时访问：

```bash
kubectl -n rook-ceph port-forward svc/rook-ceph-mgr-dashboard 7000:7000
```

管理员密码：

```bash
kubectl -n rook-ceph get secret rook-ceph-dashboard-password \
  -o jsonpath='{.data.password}' | base64 -d; echo
```

生产环境如果要长期暴露 Dashboard，应单独配置认证、TLS、Ingress 白名单或 VPN，不要直接公网暴露。

## 节点或磁盘故障

1 台节点故障时，3 副本集群通常仍可服务：

```text
MON 还剩 2/3
OSD 还剩 2/3
RGW / MDS 可切换
```

处理原则：

```text
先看 ceph status
不要反复重启所有 Pod
不要在恢复期间继续做升级
不要在 nearfull 状态下长时间大规模写入
```

磁盘替换前必须明确：

```text
哪个 OSD 对应哪块盘
是否已经 out
是否已经完成 backfill
新盘是否是裸盘
```

## 删除风险

本方案默认：

```text
CephFS preserveFilesystemOnDelete: true
RGW preservePoolsOnDelete: true
StorageClass reclaimPolicy: Retain
```

这些配置是为了避免误删 CR 后立刻删除底层数据。真正销毁 Rook Ceph 集群前，必须按 Rook 官方清理流程处理 finalizer、OSD 数据目录、磁盘签名和 CRD。

不要直接：

```bash
kubectl delete ns rook-ceph
```

这种删除方式很容易留下卡住的资源和磁盘残留。
