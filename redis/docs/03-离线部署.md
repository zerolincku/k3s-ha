# 离线部署

Redis Sentinel 离线部署需要提前准备容器镜像。K3s 本身的离线部署由 `k3s/` 目录描述，本目录只描述 Redis 资源。

## 准备镜像

在有网络的机器拉取固定版本镜像：

```bash
docker pull redis:7.2.4
docker save redis:7.2.4 -o redis-7.2.4.tar
```

把镜像包传到每台 K3s 节点：

```bash
scp redis-7.2.4.tar root@k3s-master-1:/tmp/
scp redis-7.2.4.tar root@k3s-master-2:/tmp/
scp redis-7.2.4.tar root@k3s-master-3:/tmp/
```

在每台节点导入到 K3s containerd：

```bash
k3s ctr images import /tmp/redis-7.2.4.tar
```

确认镜像存在：

```bash
k3s ctr images ls | grep 'redis:7.2.4'
```

## 使用内网 registry

如果生产环境有内网 registry，建议同步镜像后使用内网地址：

```bash
docker tag redis:7.2.4 registry.example.com/library/redis:7.2.4
docker push registry.example.com/library/redis:7.2.4
```

配置：

```bash
REDIS_IMAGE=registry.example.com/library/redis:7.2.4
```

如果 registry 需要认证，应先按 K3s 或 namespace 的镜像拉取认证方案准备 `imagePullSecret`。当前基础 manifest 默认不内置 registry 凭据。

## 执行部署

离线环境中执行同一个部署脚本：

```bash
bash redis/scripts/deploy-redis-sentinel.sh redis/prod.env
```

部署脚本不会下载任何二进制，只会通过 `kubectl apply` 创建 Kubernetes 资源。只要目标节点已经有 Redis 镜像，Pod 就可以离线启动。

## 离线检查

确认 Pod 没有尝试外部拉镜像：

```bash
kubectl -n redis describe pod redis-0 | grep -E 'Image|Pull|Err|BackOff'
```

确认 Sentinel 正常：

```bash
kubectl -n redis exec sentinel-0 -- \
  redis-cli -p 26379 sentinel masters
```
