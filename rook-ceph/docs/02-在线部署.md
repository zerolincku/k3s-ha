# 在线部署

本页描述在已有 3 master K3s 集群上在线部署 Rook Ceph、CephFS 和 RGW。

## 前置条件

每台节点必须准备独立裸数据盘，例如：

```text
k3s-master-1 /dev/vdb
k3s-master-2 /dev/vdb
k3s-master-3 /dev/vdb
```

数据盘要求：

```text
未格式化
未挂载
没有分区表
不是系统盘
不是已经被其他服务使用的盘
```

检查磁盘：

```bash
lsblk -f
blkid /dev/vdb || true
mount | grep /dev/vdb || true
```

如果磁盘曾经用过，必须先确认数据可以清空，再擦除签名：

```bash
wipefs -a /dev/vdb
sgdisk --zap-all /dev/vdb
```

## 配置

```bash
cp rook-ceph/config.example.env rook-ceph/prod.env
vim rook-ceph/prod.env
```

重点配置：

```bash
ROOK_CEPH_DEVICES=/dev/vdb
ROOK_CEPH_MON_COUNT=3
ROOK_CEPH_OBJECT_STORE_GATEWAY_INSTANCES=2
ROOK_CEPH_FS_STORAGE_CLASS=rook-cephfs
ROOK_CEPH_BUCKET_STORAGE_CLASS=rook-ceph-bucket
```

如果每台节点有多块数据盘：

```bash
ROOK_CEPH_DEVICES=/dev/vdb,/dev/vdc
```

生产不建议设置：

```bash
ROOK_CEPH_USE_ALL_DEVICES=true
```

显式指定数据盘更安全，避免误把系统盘或临时盘纳入 Ceph。

## 部署

```bash
bash rook-ceph/scripts/deploy-rook-ceph.sh rook-ceph/prod.env
```

脚本执行顺序：

```text
1. 安装 Rook CRD / RBAC / CSI / Operator
2. 创建 CephCluster
3. 等待 CephCluster Ready
4. 创建 CephFilesystem 和 rook-cephfs StorageClass
5. 创建 CephObjectStore、rook-ceph-bucket StorageClass 和 RGW 用户
6. 输出 Pod、CR、StorageClass、RGW endpoint 状态
```

## 检查

查看 Pod：

```bash
kubectl -n rook-ceph get pods -o wide
```

查看 Ceph CR：

```bash
kubectl -n rook-ceph get cephcluster,cephfilesystem,cephobjectstore,cephobjectstoreuser
```

查看 StorageClass：

```bash
kubectl get storageclass | grep rook-ceph
```

预期存在：

```text
rook-cephfs
rook-ceph-bucket
```

查看 RGW Service：

```bash
kubectl -n rook-ceph get svc | grep rook-ceph-rgw
```

集群内 S3 endpoint：

```text
http://rook-ceph-rgw-objectstore.rook-ceph.svc:80
```

## 获取 RGW 用户密钥

默认用户：

```text
CephObjectStoreUser: admin
Secret: rook-ceph-object-user-objectstore-admin
```

读取：

```bash
kubectl -n rook-ceph get secret rook-ceph-object-user-objectstore-admin \
  -o jsonpath='{.data.AccessKey}' | base64 -d; echo

kubectl -n rook-ceph get secret rook-ceph-object-user-objectstore-admin \
  -o jsonpath='{.data.SecretKey}' | base64 -d; echo
```

## 创建业务 Bucket

推荐业务目录通过 ObjectBucketClaim 动态创建 bucket：

```yaml
apiVersion: objectbucket.io/v1alpha1
kind: ObjectBucketClaim
metadata:
  name: app-bucket
  namespace: default
spec:
  generateBucketName: app
  storageClassName: rook-ceph-bucket
```

应用后：

```bash
kubectl apply -f app-bucket.yaml
kubectl get obc -A
```

OBC 会生成对应的 ConfigMap 和 Secret，业务应用从其中读取 endpoint、bucket、access key 和 secret key。
