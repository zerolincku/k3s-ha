# Keepalived + HAProxy 架构

## 目标

提供一个可漂移的 TCP VIP 入口，当前用于 K3s apiserver：

```text
client -> VIP:8443 -> HAProxy TCP frontend -> backend:6443
```

## 组件职责

- Keepalived：通过 VRRP 在多台节点之间漂移 VIP。
- HAProxy：监听 VIP 端口，将 TCP 流量转发到健康后端。
- VIP：客户端、kubectl、K3s 节点加入使用的稳定地址。

## K3s 场景的端口

| 端口 | 用途 |
| --- | --- |
| `8443/tcp` | VIP 对外入口 |
| `6443/tcp` | K3s apiserver 后端端口 |

当 HAProxy 与 K3s master colocated 时，外部入口使用 `8443`，避免 HAProxy 与 K3s apiserver 争用本机 `6443`。
