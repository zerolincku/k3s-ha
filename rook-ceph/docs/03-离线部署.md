# 离线部署

Rook Ceph 离线部署需要准备：

```text
Rook 官方 manifest
Rook Operator 镜像
Ceph 镜像
Ceph CSI 相关镜像
CSI sidecar 镜像
```

默认 Rook Operator 镜像使用：

```text
quay.io/rook/ceph:v1.19.6
```

Rook 官方示例 manifest 中的 Operator 镜像可能是 `docker.io/rook/ceph:v1.19.6`。本目录默认改用 Quay，是为了减少 Docker Hub / CloudFront 在国内网络下出现 `EOF` 的概率。部署脚本会把 Operator Deployment patch 成 `ROOK_CEPH_IMAGE` 指定的镜像。

K3s 本身离线部署由 `k3s/` 目录描述，本目录只描述 Rook Ceph 资源。

## 下载离线资源

在有网络的机器执行：

```bash
cp rook-ceph/config.example.env rook-ceph/prod.env
vim rook-ceph/prod.env

bash rook-ceph/scripts/download-rook-ceph-assets.sh rook-ceph/prod.env
```

默认输出：

```text
rook-ceph/artifacts/manifests/crds.yaml
rook-ceph/artifacts/manifests/common.yaml
rook-ceph/artifacts/manifests/csi-operator.yaml
rook-ceph/artifacts/manifests/operator.yaml
rook-ceph/artifacts/images/*.tar
```

按 `linux/arm64`、Rook `v1.19.6`、Ceph `v19.2.3` 估算，镜像归档大约 5.6GiB：

```text
Ceph 主镜像：约 1.3GiB
Ceph CSI 镜像：约 2.1GiB
Rook Operator 镜像：约 1.5GiB
CSI sidecar 镜像：约数百 MiB
```

上传到 3 台 K3s 节点后，还会被 containerd 解包，占用会明显大于 tar 原始大小。验证环境每台节点系统盘建议预留 80GiB 以上。

如果需要代理，先导出：

```bash
export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890
export all_proxy=socks5://127.0.0.1:7890
```

默认使用 Docker 拉取镜像并导出 tar。如果 Docker daemon 访问 registry 不稳定，可以改用 `crane`：

```bash
GOBIN=/tmp/k3s-ha-bin go install github.com/google/go-containerregistry/cmd/crane@latest

ROOK_CEPH_IMAGE_DOWNLOAD_TOOL=crane \
CRANE_BIN=/tmp/k3s-ha-bin/crane \
bash rook-ceph/scripts/download-rook-ceph-assets.sh rook-ceph/prod.env
```

如果本地代理会替换 TLS 证书，且你确认代理可信，可以临时设置：

```bash
ROOK_CEPH_CURL_INSECURE=true \
bash rook-ceph/scripts/download-rook-ceph-assets.sh rook-ceph/prod.env
```

## 上传到 K3s 自动导入目录

K3s 会从每个节点的以下目录导入镜像归档：

```text
/var/lib/rancher/k3s/agent/images/
```

配置节点：

```bash
ROOK_CEPH_NODE_SSH_HOSTS=k3s-master-1,k3s-master-2,k3s-master-3
ROOK_CEPH_NODE_NAMES=k3s-master-1,k3s-master-2,k3s-master-3
SSH_USER=root
```

上传镜像：

```bash
bash rook-ceph/scripts/upload-rook-ceph-images.sh rook-ceph/prod.env
```

上传脚本会先把文件传成 `.uploading-*.tmp`，传输完成后再移动成最终 `.tar` 文件。不要手工把正在传输的半成品直接写入最终文件名，否则 K3s 启动扫描 images 目录时可能读到未完成 tar，出现 `short read` 或 `unexpected EOF`。

如果 K3s 已经在运行，只把 tar 放进 images 目录不会立即完成导入。需要重启对应节点的 K3s 服务，才会立刻触发自动导入。生产环境建议逐台滚动重启：

```bash
ROOK_CEPH_RESET_K3S_IMAGE_CACHE=true \
ROOK_CEPH_RESTART_K3S_AFTER_IMAGE_UPLOAD=true \
bash rook-ceph/scripts/upload-rook-ceph-images.sh rook-ceph/prod.env
```

## 离线部署

配置本地 manifest：

```bash
ROOK_CEPH_CRDS_MANIFEST=rook-ceph/artifacts/manifests/crds.yaml
ROOK_CEPH_COMMON_MANIFEST=rook-ceph/artifacts/manifests/common.yaml
ROOK_CEPH_CSI_OPERATOR_MANIFEST=rook-ceph/artifacts/manifests/csi-operator.yaml
ROOK_CEPH_OPERATOR_MANIFEST=rook-ceph/artifacts/manifests/operator.yaml
```

执行：

```bash
bash rook-ceph/scripts/deploy-rook-ceph.sh rook-ceph/prod.env
```

部署脚本不会下载二进制。只要节点已导入所有 Rook、CSI、Ceph 镜像，Pod 就可以离线启动。

部署脚本会在应用 Rook Operator manifest 后执行：

```text
kubectl -n rook-ceph set image deployment/rook-ceph-operator rook-ceph-operator=$ROOK_CEPH_IMAGE
kubectl -n rook-ceph patch deployment rook-ceph-operator ... imagePullPolicy=$ROOK_CEPH_IMAGE_PULL_POLICY
kubectl -n rook-ceph set env deployment/rook-ceph-operator ROOK_CSI_IMAGE_PULL_POLICY=$ROOK_CEPH_IMAGE_PULL_POLICY
```

因此离线资源中的 Rook Operator 镜像必须和 `ROOK_CEPH_IMAGE` 一致。
`ROOK_CEPH_IMAGE_PULL_POLICY` 默认是 `IfNotPresent`，语义是优先使用 K3s 已从 `/var/lib/rancher/k3s/agent/images/` 导入的本地镜像；如果希望完全禁止节点尝试外部拉取，可以设置为 `Never`。

## 离线检查

确认镜像已导入：

```bash
k3s ctr images ls | grep -E 'rook|ceph|csi'
```

确认 Pod 没有外部拉镜像失败：

```bash
kubectl -n rook-ceph get pods -o wide
kubectl -n rook-ceph describe pod <pod-name> | grep -E 'Image|Pull|Err|BackOff'
```

如果某个 CSI sidecar 镜像缺失，重新执行下载脚本。下载脚本会从 Rook 官方 operator manifest 中提取镜像列表，正常情况下不需要手工维护 CSI 镜像清单。
